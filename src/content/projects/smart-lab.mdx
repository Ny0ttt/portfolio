## The Problem

The SEiR research lab at MJIIT had no centralized facility management. Door access existed, but there was no PC usage tracking, no CCTV integration, and no unified view of who was using what.

## The Solution

A microservice portal that ties together three systems: door access control (ZKBio CVSecurity), CCTV with AI object detection (Frigate+), and a custom-built Windows Forms computer access module.

## My Role

Built the computer access software, the AI object detection layer, and the RESTful APIs connecting everything. Responsible for the full SDLC — requirements gathering, UI/UX, coding, and testing. Used an Agile approach to accommodate stakeholder feedback throughout.

## Stack

- **Backend**: .NET Core, RESTful APIs
- **Database**: MySQL
- **Infrastructure**: Docker Compose
- **CCTV / AI**: Frigate NVR, Gemini API, Ollama (Mistral — runs locally)
- **Architecture**: Microservice

## Results

- 95% UAT pass rate
- All requirements met and stakeholder-approved
- Demonstrated live integration of AI object detection in a real lab environment

## Key Challenges

- Integrating three independently-designed systems (ZKBio, Frigate, and the custom Windows Forms app) into one coherent portal without modifying the existing door access infrastructure
- Running AI object detection locally with Ollama/Mistral to avoid cloud dependency — required tuning inference performance within lab hardware constraints
- Agile stakeholder management across academic and technical audiences with differing expectations
